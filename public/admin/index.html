API 使用文档

您可以通过以下标准 HTTP API 接口，让外部脚本或 AI 程序直接操作本系统绑定的微软邮箱，实现邮件的自动化收发。这些接口直接复用了本系统的 api_key 鉴权机制。

1. 读取最新邮件 (直连查询)

功能： 绕过公用链接验证，直接根据邮箱地址拉取最新的收件箱和垃圾箱邮件。

请求方式： GET 或 POST

接口路径： /api/direct/read

参数说明：

  • api_key (必需): URL 传参，填写您在【系统设置】中配置的 API Key。
  • email (必需): 您想要读取的目标微软邮箱地址（必须是已添加进本系统的邮箱）。
  • limit (可选): 限制返回的邮件数量，默认只获取最新 1 封。如果是3封：https://您的域名/api/direct/read?api_key=系统设置里的秘钥&email=test@outlook.com&limit=3

GET 调用示例 (cURL)：

curl -X GET "https://您的域名/api/direct/read?api_key=系统设置里的秘钥&email=test@outlook.com"

返回格式示例 (JSON)：

{
  "ok": true,
  "data": [
    {
      "id": "AAMkAGUzY...",
      "subject": "邮件主题",
      "sender": "发件人名称 <sender@example.com>",
      "receiver": "test@outlook.com",
      "htmlContent": "<html>正文内容...</html>",
      "body": "纯文本预览...",
      "received_at": "2026-07-01T10:00:00Z"
    }
  ]
}
2. 发送新邮件 (直连发信)

功能： 指定系统中的某个微软账号，向外部邮箱发送邮件。非常适合 AI 自动回复或通知脚本调用。

请求方式： POST

接口路径： /api/direct/send

参数说明 (必须放在请求体 JSON 中)：

  • api_key (必需): 依旧放在 URL 传参中。
  • email (必需): 系统内已保存的发件邮箱地址。
  • to (必需): 目标收件人邮箱。
  • subject (必需): 邮件主题。
  • content (必需): 邮件正文，支持 HTML。

POST 调用示例 (cURL)：

curl -X POST "https://您的域名/api/direct/send?api_key=系统设置里的秘钥" \
  -H "Content-Type: application/json" \
  -d '{
    "email": "发件箱@outlook.com",
    "to": "目标收件人@example.com",
    "subject": "AI 自动发送测试",
    "content": "<h1>这是一封通过 API 直连发送的邮件</h1>"
  }'

返回格式示例 (JSON)：

{
  "ok": true,
  "error": null
}
添加邮箱
请按顺序粘贴，使用逗号或竖线分隔：
Client ID | Client Secret | Refresh Token
批量导入邮箱

格式：邮箱地址 [TAB] 密码 [TAB] Client ID [TAB] Client Secret [TAB] Refresh Token
每行一个账号，密码和Secret不填可留空（需保留TAB空格占位）：

批量添加任务 (导入)

格式：发件邮箱 [TAB] 收件人 [TAB] 主题 [TAB] 内容 [TAB] 延迟配置 [TAB] 循环(0/1) [TAB] 执行时间
说明：以制表符分隔，空值保留占位。发件邮箱需匹配已有账号。
